## 一、用户角色定义

| 角色 | 描述 | 典型用户 | 核心目标 |
|---|---|---|---|
| **Agent 开发者** | 构建和部署 AI Agent 的工程师 | DeFi 协议团队、量化交易团队 | 让 Agent 安全持有私钥并自动交易 |
| **Agent 使用者** | 使用 AI Agent 管理资产的个人或机构 | 加密资产持有者、DAO 财库管理者 | 放心委托 Agent 操作资金，不被异常交易损失 |
| **协议集成者** | 将 Chiron 集成到自己的 DApp/钱包的开发者 | DeFi 协议、钱包团队、安全工具开发者 | 为协议用户提供 Agent 交易安全保障 |
| **社区贡献者** | 扩展 Chiron 生态的非核心开发者 | 安全研究员、协议爱好者 | 提交新协议映射、检测规则、攻击测试 |
| **安全审计者** | 审核 Chiron 安全的第三方 | 审计公司、白帽黑客 | 验证 Chiron 能有效防范声称的攻击 |

## 二、用户故事清单

### 2.1 Agent 开发者的故事

#### US-01 结构化意图声明

> **作为** Agent 开发者
> **我希望** 用标准格式声明 Agent 的交易意图（Swap/Bridge/Deposit 等）
> **以便** Chiron 能自动验证实际执行是否匹配声明

**验收条件：**

| # | 条件 | 优先级 |
|---|---|---|
| 1.1 | 提供 `chiron.intent(action, params)` API 生成 IntentTemplate | P0 |
| 1.2 | 支持 ≥ 9 种 Action 类型（swap, approve, bridge, deposit, withdraw, stake, vote, permit, custom）| P0 |
| 1.3 | IntentTemplate 序列化为 bytes32 hash，可链上存证 | P0 |
| 1.4 | 自动填充 token 地址（symbol → address 转换）| P1 |
| 1.5 | 自动处理 decimal 精度转换（如 USDC 6 decimals → wei）| P1 |

**示例：**

```typescript
// Agent 开发者写这 1 行
const intent = chiron.intent("swap", {
  protocol: "uniswap_v3",
  tokenIn: "USDC",
  tokenOut: "WETH",
  amount: "100",
});
```

### 2.2 Agent 使用者的故事


## 五、关键技术决策验证

### 5.1 结构化 Intent 而非自然语言 —— 验证

**假设：** 结构化模板足够表达 Agent 的交易意图，不需要自然语言。

**验证方式：**
- [ ] 收集 100 笔真实 Agent 交易 → 能否用结构化模板表达
- [ ] 如果 > 90% 可表达 → 假设成立
- [ ] 不可表达的 10%：是否可以通过新增 Action 类型或 `custom` 兜底解决

**风险：** 复杂策略（如「如果 ETH 价格在 1800-2000 之间就买，否则等一天」）无法用结构化模板表达 → 需使用 `custom` 类型兜底，该类交易仅做基本检查。

### 5.2 L1 覆盖 95% —— 验证

**假设：** 确定性本地校验可覆盖 95% 的 Agent 交易场景。

**验证方式：**
- [ ] 收集主流 DeFi 协议（Uniswap, Curve, Aave, Compound, Balancer, Lido 等）的交易类型
- [ ] 对照 IntentTemplate Action 类型检查覆盖率
- [ ] 如果覆盖率 < 90% → 新增 Action 类型或放宽 L1→L2 升级条件

## 六、验收测试用例

### 6.1 US-01 验收测试

```typescript
// TC-01: 标准 Swap 意图生成
const intent = chiron.intent("swap", {
  protocol: "uniswap_v3",
  tokenIn: "USDC",
  tokenOut: "WETH",
  amount: "100",
});
assert(intent.actionType === 1);                         // SWAP
assert(intent.protocolHash === keccak256("uniswap_v3"));
assert(intent.amount === "100000000");                    // 6 decimals → wei

// TC-02: 非法 Action 类型
assert.throws(() => chiron.intent("invalid_action", {}));

// TC-03: 意图 hash 确定性
const hash1 = chiron.intent("swap", {...}).hash();
const hash2 = chiron.intent("swap", {...}).hash();
assert(hash1 === hash2);
```

### 6.2 US-02 验收测试

```typescript
// TC-04: 3 行集成
// 用户原始代码:
const tx = buildSwapTx(amount, recipient);
// 新增的 3 行:
const receipt = await chiron.verify(intent, tx);
if (!receipt.allowed) throw new Error(receipt.reason);
// 用户原始代码继续:
await wallet.sendTransaction(tx);

// TC-05: 无需链上交互的 L1 校验
const receipt = await chiron.verify(intent, tx);
assert(receipt.l1Result !== undefined);
assert(receipt.gasUsed === undefined);  // 无链上交互
```

### 6.3 US-04 验收测试

```typescript
// TC-06: 正常交易放行
const receipt = await chiron.verify(
  intent("swap", { protocol: "uniswap_v3", tokenIn: "USDC", amount: "100" }),
  buildSwapTx(/* 正常参数 */)
);
assert(receipt.allowed === true);

// TC-07: 不一致交易阻止
const receipt = chiron.verify(
  intent("swap", { protocol: "uniswap_v3", ... }),
  buildApproveTx(/* 给攻击者的 Approve */)
);
assert(receipt.allowed === false);
assert(receipt.reason.includes("swap"));

// TC-08: 金额异常阻止
const receipt = chiron.verify(
  intent("swap", { amount: "100" }),
  buildSwapTx({ amount: "10000" })
);
assert(receipt.allowed === false);
assert(receipt.reason.includes("amount"));
```

### 6.4 US-06 验收测试

```typescript
// TC-09: 链上存证查询
await chiron.storeReceipt(receipt);
const stored = await chiron.getReceipt(receipt.txHash);
assert(stored.intentHash === receipt.intentHash);
assert(stored.l1Result === receipt.l1Result);

// TC-10: Agent 历史查询
const history = await chiron.getAgentReceipts(agentAddress);
assert(history.length >= 1);
assert(history[0].txHash === receipt.txHash);
```

### 6.5 US-07 验收测试

```typescript
// TC-11: CircuitBreaker 暂停
for (let i = 0; i < 5; i++) {
  await chiron.verify(intent, maliciousTx);    // 5 次 FAIL
}
const agentStatus = await circuitBreaker.getAgentStatus(agentAddress);
assert(agentStatus.paused === true);
assert(agentStatus.reason === "5 consecutive verification failures");

// TC-12: 暂停后恢复
await circuitBreaker.resume(agentAddress, userSignature);
const status = await circuitBreaker.getAgentStatus(agentAddress);
assert(status.paused === false);

// TC-13: 日额度超限暂停
for (let i = 0; i < 1001; i++) { /* 触发日交易上限 */ }
const status = await circuitBreaker.getAgentStatus(agentAddress);
assert(status.paused === true);
assert(status.reason === "Daily limit exceeded");
```
