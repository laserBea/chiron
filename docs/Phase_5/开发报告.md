## 二、核心模块

### 2.1 BatchParser + BatchVerifier

**用途：** 解析 ERC-8211 批量交易的 calldata，对每步独立执行 L1 校验。

```
BatchVerifier.verify(batchIntent, batchTx)
    │
    ├─ BatchParser.parseSteps(tx.data) → BatchStep[]
    │
    ├─ 每步执行：
    │   ConsistencyChecker.check(stepIntent, stepTx)
    │   ↓
    │   StepResult { allowed, l1Result, reason, details }
    │
    ├─ aggregateResult = any(step FAIL) ? 'FAIL' : 'PASS'
    │
    └─ BatchVerificationResult { aggregateResult, steps[] }
```

**支持批量类型：**
- `0x5ae401dc` — Uniswap V3 Multicall
- `0xac9650d8` — Uniswap V3 Multicall (alternative)
- 自定义 batch — 按 4-byte selector 边界分割

### 2.2 IntentTemplateRegistry

**用途：** 社区在链上注册新协议 → Action 映射。

```
struct ProtocolMapping {
    bytes32 protocolHash;
    address[] contracts;
    bytes4[] selectors;
    uint8 actionType;
    bool active;
    uint256 registeredAt;
}
```

**接口：**

| 函数 | 说明 |
|---|---|
| `registerProtocol(hash, contracts, selectors, actionType)` | 注册新协议 |
| `deactivateProtocol(hash)` | 停用协议（Owner only）|
| `getActionType(address)` | 查询合约对应 Action |
| `getRegisteredContractCount()` | 统计已注册合约数 |

### 2.3 Webhook 告警

**用途：** L1 FAIL 事件 → 推送到用户配置的 URL。

```
Chiron.verify() → FAIL
    │
    ▼
Webhook Service → POST /alert
    │
    ├─ event: "FAIL"
    ├─ agent: 0xAgentAddress
    ├─ txHash: 0x...
    ├─ reason: "R1: target not in registry"
    └─ pauseAgentUrl: https://chiron.agent/pause/0x...
```

**配置：**

```json
POST /register
{ "id": "my-bot", "url": "https://my-server.com/chiron-alert",
  "secret": "hmac-secret", "events": ["FAIL", "PAUSED"] }
```
