# Chiron 开发计划（用户故事驱动版）

> 版本：v2.0
> 日期：2026-06-11
> 组织方式：按用户故事垂直切分，每个阶段交付完整的端到端能力

---

## 一、设计原则

### 1.1 垂直切片（Vertical Slicing）

每个阶段交付**一个完整的用户能力**，而非一个技术层。用户在每个阶段末尾都能获得可用的价值。

```
传统方式：        用户故事驱动方式：
                   ┌────────────────────────────┐
  Phase 1: SDK     │ Phase 1: US-01 + US-02     │ ← Agent 可声明意图并校验
  Phase 2: 合约    │   → IntentParser            │
  Phase 3: 测试    │   → TxDecoder               │
                   │   → ConsistencyChecker       │
                   │   → 3 行集成 API            │
                   │   → L1 校验可跑通           │
                   └────────────────────────────┘
                   ┌────────────────────────────┐
                   │ Phase 2: US-04 + US-06     │ ← 使用者可放心委托 + 查审计
                   │   → ReceiptBuilder          │
                   │   → VerificationStore 合约  │
                   │   → 链上存证 + 查询          │
                   │   → Demo Agent              │
                   └────────────────────────────┘
```

### 1.2 每个阶段的交付标准

每个 Phase 结束时必须满足：

- [ ] 该阶段覆盖的用户故事全部验收通过
- [ ] 对应的验收测试用例（TC）全部绿
- [ ] 新增代码集成到 CI 流水线
- [ ] 文档同步更新

### 任务分解

| 天 | 任务 | 产出 | 验证 |

| 3 | **Calldata 解码器** | `decoder.ts` — 接收 target + data → DecodedTx<br>4byte selector → 函数名<br>ABI 参数解析 | 正确解码 Uniswap Swap<br>正确解码 Approve<br>未知 selector 给出 fallback |
| 4 | **一致性校验规则 1-3** | `checker.ts` — 合约白名单检查<br>函数签名检查<br>参数类型检查 | TC-07: Approve 替 Swap → FAIL<br>TC-08: 金额异常 → FAIL |
| 5 | **一致性校验规则 4-6** | `checker.ts` — 金额 ±slippage 检查<br>异常参数检查<br>嵌套 delegatecall 检测 | 恶意 delegatecall → FAIL<br>正常滑点 → PASS |
| 6 | **校验规则测试** | 6 条规则的 Jest 测试用例 | TC-06: 正常 → PASS<br>TC-07: 不一致 → FAIL<br>TC-08: 金额超限 → FAIL |
| 7 | **内置协议地址表** | `registry.ts` — ≥ 20 个主流合约地址 → Action 映射<br>Uniswap V2/V3, Curve, Balancer, Aave, Compound, Lido, 1inch 等 | 查询已知协议 → 返回 Action<br>查询未知合约 → 返回 UNKNOWN |
| 8 | **SDK 主入口集成** | `index.ts` — `chiron.intent()` + `chiron.verify()` + `chiron.storeReceipt()`<br>3 行集成 API 定型 | TC-04: 3 行集成验证<br>TC-05: 无链上交互的 L1 校验 |
| 9 | **集成测试** | 端到端测试：Agent 声明意图 → 构建交易 → L1 校验 → 通过/阻断 | 5 条端到端测试全部通过 |
| 10 | **Buffer + 文档** | README 示例更新<br>已知问题清单<br>API 文档骨架 | — |
