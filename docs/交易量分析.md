# L1 注册表交易量分析

> 目的：验证当前 Protocol Registry 的协议覆盖能否达到 L1 校验 95% 覆盖率目标，找出必须补的协议。

---

## 一、数据来源与方法

### 来源
- **DeFi Llama**: 各协议 TVL（总锁仓价值），2026-06 实时数据
- **The Block**: 30 天 Gas 消耗 Top 20 合约
- **Etherscan**: 合约调用频率

### 代理交易量的估算方法

由于缺乏直接的"AI Agent 调用统计"，用以下指标综合估算代理交易占比：

```
代理交易活跃度 ≈ TVL(协议) × Agent 兼容度系数

Agent 兼容度系数：
  1.0 — 可完全由 Agent 自动化操作的协议（DEX、借贷、质押）
  0.7 — 半自动协议（跨链桥、收益聚合）
  0.3 — 被动型协议（RWA、稳定币铸造，Agent 操作较少）
```

## 五、优先级建议

### P0 — 必须补（Agent 最常交互，覆盖 28% → 65%）

| 协议 | 原因 | 工作 |
|---|---|---|
| **Lido** | TVL 第一，Agent 频繁做质押/兑换 stETH | 加 stETH, wstETH 合约 + submit/withdraw |
| **Morpho** | TVL 第三，借贷领域增长最快，Agent 最活跃 | 加 Morpho 合约 + supply/withdraw |
| **Cow Swap** | Intent 原生协议，Agent-to-Agent 交易的核心 | 加 Cow Swap 合约 + swap 相关 selector |
| **Spark** | TVL 第六，与 Maker 深度绑定，DAI 铸造入口 | 加 Spark 合约 + supply/withdraw/borrow |
| **Sky/MakerDAO** | Agent 做 DAI 铸造/储蓄的核心入口 | 加 DSR 合约 + sDAI 相关 |

### P1 — 尽快补（覆盖 65% → 80%）

| 协议 | 原因 |
|---|---|
| **Ethena** | sUSDe 是 Agent 最常用的收益资产之一 |
| **EigenLayer** | 再质押赛道龙头，Agent 做 LST → 再质押的常见路径 |
| **ether.fi** | eETH 是第二大流动性质押代币 |
| **Pendle** | 收益代币化，Agent 做高级收益策略的必经协议 |
| **Stargate** | 被动跨桥中最常用的协议 |

### P2 — 按需补（覆盖 80% → 90%）

| 协议 | 原因 |
|---|---|
| Binance stETH | TVL 大但主要在 BSC |
| Convex | Agent 做 Curve 收益 booster |
| Yearn | 收益聚合，Agent 自动再平衡的终点 |
| RocketPool | 去中心化质押，rETH 是重要 LST |
| Circle CCTP | 原生 USDC 跨链，Agent 跨链的标准路径 |

### P3 — 长尾（90% → 95%）

各 L2 的本地龙头协议（Velodrome、Aerodrome、GMX 等），单协议 TVL 通常 < 1B。

## 六、如果补完 P0 的效果

```
当前覆盖率:                28%
+P0(5 个协议):  Lido(14.71) + Morpho(6.43) + Cow Swap(0.80) + Spark(4.73) + Sky(4.16) = 30.83
新总覆盖权重:   67.86 + 30.83 = 98.69
新覆盖率:       (18.82 + 30.83) / 98.69 = 50.3%
```

补完 P0 只能到 50%。还需要 P1 才能到 65% 左右。要达到 95%，需要 P0+P1+P2 全部补上，再加 L2 本地协议。

**结论：95% 的 L1 覆盖率需要约 40-50 个协议注册。当前 14 个远远不够，至少需要补充 P0+P1 共约 12 个协议才能达到 65%。**
